2026-04-27 草川たくやのNotionワークスペースに、ミーティングノートを会議種別ごとに整理する2層DBシステムを構築。

**ID群**:
- 親（会議ハブ）: `a247fd5d-56da-4acd-b9db-3ad97cec6a12`
- 📅 ミーティングノートDB: data_source=`26d7848d-ad1a-4e84-806f-a8dbccb1872b`、database=`f973bf6c-82da-44c1-a517-0fcf12a03d85`
- 🏛 会議体マスタDB: data_source=`46414643-9717-404f-9572-2908c2c0a7f3`、database=`66ce3b03-04ac-474a-86cb-c81742ed9030`
- AIミーティングノートDB: `58ecf503-a68f-83d4-aa9e-8144cabd7b8b`（**system collection、relation不可・読み取り専用**）

**スキーマ要点**:
- 会議種別: 議会公務／地域・団体／市民相談／その他／未分類（4＋1）
- 📅DBは18プロパティ、5DB(タスク/政策候補/市民相談/質問ネタ/会議体マスタ)へDUAL relation。AI元ノートはURL型でフォールバック
- 🏛マスタは10プロパティ＋ROLLUP「開催回数」(関連ミーティング count)

**会議体マスタ既登録4件（IDマッピング）**:
- 亀山市eスポーツ協会: `34fcf503-a68f-81d7-b064-ffb9dc2ed54b`
- 亀山JC（青年会議所）: `34fcf503-a68f-811d-b35b-e50bf49fd837`
- コスモス三寺: `34fcf503-a68f-81a4-9115-d178770befd8`
- 三重大学応援団: `34fcf503-a68f-8114-a553-d790be383d38`

**Why**: 草川のミーティング系記録がAIミーティングノートDB／散在ページ／会議ハブの3箇所に分散していて「来週どの会議体がフォローアップ必要か」「政策候補と紐づく会議履歴は何か」が即座に見えない問題への対応。

**How to apply**:
- 新規ミーティング → 📅DBに直接登録、種別タグ＋会議体relation（既存マスタとマッチすれば）
- 継続会議体（自治会・協会・委員会等）→ 🏛マスタに先に登録
- AIミーティングノートDBはNotion AI管理下のため schema変更不可。📅DB側に「AI元ノートURL」プロパティで紐付け
- nichijoスキル整理モード(R4-A)で当日新規ミーティング自動振り分け
- 「未分類」レコードは週次/月次棚卸しで4種別に振り分け
