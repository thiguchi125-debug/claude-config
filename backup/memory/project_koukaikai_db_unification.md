# 後援会員DB統合（2026-05-04）

## 経緯

旧設計:
- **後援会員DB**(15deb49e-) = 確定名簿用（KAI-{n}・正会員/準会員/支援者/関係者の区分）
- **役員・サポートメンバーページ**(2ddcf503-a68f-80bd-a3f5-c9aa7e2d33d5) = 73名の打診管理（顧問8+幹事20+サポート15+ウグイス27+事務局2+SNS1）

問題: 承諾もらった重要人物が DB に反映されない／役員ページの中の人とDB会員が分離して把握困難。

草川判断「ページを分ける必要がない。役員ページの中で承諾もらった人を後援会員DBへ記録していくイメージ」→ **1DB統合**。

## DB拡張内容

- **役職** select: 顧問/幹事/サポートメンバー/ウグイス/事務局/SNSチーム/会員のみ
- **打診ステータス** select: 未着手/打診中/承諾済/既任/辞退/離任
- **打診担当** text
- **打診日** date
- **承諾日** date
- **自治会** relation → 自治会別訪問管理DB（DUAL）
- 旧「9地区訪問リンク」(廃止DB参照) → DROP
