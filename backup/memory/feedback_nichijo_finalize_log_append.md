仕上げモード（nichijo統合仕上げ・U1〜U4）の処理後、生成・上書きしたコンテンツや軌道修正の経緯が日次ルーティンログDBの当日「YYYY-MM-DD 活動ログ」へ自動追記されない仕様の穴がある。U4サマリー出力後に必ず日次ログへの追記処理を実行する。

**Why:** 2026-04-29仕上げで、太陽光条例案検証のブログ深掘り＋SNS5媒体を生成し、当初前提「届出制」→PDF実物確認後「許可制」への重要な軌道修正を行ったが、これらが当日活動ログに自動反映されず、ユーザー指摘で初めて気づいた。日次ログは1日の活動の起点・統合ビューであり、生成コンテンツへのリンクや軌道修正の学びが残らないと、後日の振り返り・週次レポート（oyasumi金曜）・voice-dna整合性チェックが弱くなる。

**How to apply:**
- 仕上げモード（統合「仕上げ」単独トリガーで起動するU1〜U4）の最終ステップとして、U4サマリー出力後に日次ルーティンログDB（data_source_id: f6d15ac0-73fa-414f-8242-03e8f06aa4e3）の当日「YYYY-MM-DD 活動ログ」を fetch → notion-update-page の update_content で末尾に「### 追記 [仕上げ / {テーマ}]」セクションを追記する
- 追記内容には(1)生成物のNotion URL（ブログ・SNS・タスク）、(2)Part A/Bで起きた重要な事実修正・軌道修正の経緯、(3)後日参照したい論点リストを必ず含める
- 当日活動ログが存在しない場合は新規作成（2-Cと同じ手順）してから追記
- 単独モード（仕上げモード／コンテンツ抽出モード／整理モード単独）でも同様に追記する
- nichijoスキル定義のU4直後にこの手順を組み込む改修が必要（恒久対応）
