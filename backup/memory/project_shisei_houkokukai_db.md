# 🎤 市政報告会DB（2026-05-04新設）

## 経緯

訪問前に各自治会で市政報告会を開催し、案内＋地域版チラシを配布する運用を継続中（5月も継続実施。「6月から本格実施」は誤認識→修正済）。これらの報告会記録を「記録: 市政報告会 〇〇自治会」形式でnichijoから自動DB化したい要望。

## DB

- **DB URL**: https://www.notion.so/1338a7e9dedd4616aa19c2e2559fee26
- **collection_id**: `df08b5ea-f5ff-4022-afe2-c8073fbe341a`
- **親**: 🗺️ 地域・選挙ハブ
- **報告会ID**: `SHK-{n}` 自動採番

## スキーマ（19フィールド）

### 基本情報
- タイトル (title)
- 開催日 (date)
- 開催地区 (select: 22地区＋複数地区合同＋市外)
- 開催自治会 (relation → 自治会別訪問管理DB DUAL)
- 形態 (select: 自治会単位/複数自治会合同/地区全体/オンライン主/ハイブリッド/街頭ミニ)
- 会場 (rich_text)

### 開催実績
- 参加者数 (number)
- 質問件数 (number)
- 主な意見・要望 (rich_text)
- 議会で取り上げる課題 (rich_text)
- 進捗ステータス (status: 未着手/進行中/完了)

### 配布物・記録
- 配布チラシ (rich_text)
- 動画URL (url)
- 写真URL (url)
- メモ (rich_text)

### 関連
- 関連タスク (relation → ✅ タスクDB DUAL)
- 関連市民意見 (relation → 📝 市民意見リスト DUAL)
- 報告会ID (auto SHK-{n})

## ビュー（5本）

- 📅 カレンダー — CALENDAR BY 開催日
- 🎤 開催済 — FILTER 進捗ステータス=完了, SORT 開催日 DESC
- 📋 計画中・告知中 — SORT 開催日 ASC
- 🗺️ 地区別ボード — GROUP BY 開催地区
- 📊 全レコード — SORT 開催日 DESC

## nichijo連動

「記録: 市政報告会 〇〇自治会 5/16 参加18人 質問8件 主な意見=みずきが丘の通学路、水道濁り...」
→ 自動でレコード作成・関連自治会(relation)を紐付け・進捗ステータス=完了で書込

## 自治会別訪問管理DBとの違い

| | 🏘️ 自治会別訪問管理DB | 🎤 市政報告会DB |
|---|---|---|
| 単位 | 自治会1レコード | 開催1回1レコード（複数回可） |
| 用途 | 訪問進捗・関係性温度・累積指標 | 個別報告会の参加者・意見・配布記録 |
| 例 | 「みずきが丘」=1レコード | 「みずきが丘 5/16第1回」「みずきが丘 6/20第2回」=2レコード |

両DBはDUAL relationで連携（開催自治会フィールド）。
