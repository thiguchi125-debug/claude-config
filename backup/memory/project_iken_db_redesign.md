## 新設計（2026-05-02 確定）

### モデル

**📝 市民意見リスト(`c2c34bd8-`)** = 唯一の永続意見ログ
- 市民の声をそのまま記録、消さない、ステータスで埋もれさせない
- `対応状況` フィールドは残るが workflow 駆動はしない（参考情報のみ）
- タスク化が必要なものは **「関連タスク」relation** で✅タスクDBへ抽出
- タスクDB側で進捗管理（KAI-{n}番号で追跡）

### スキーマ追加（c2c34bd8-）
- 緊急度 (select 高/中/低)
- 地区 (select 11区分)
- 受付ID (UNIQUE_ID prefix 'IKN')
- 関連タスク (relation to タスクDB, DUAL '関連市民意見')

## 設計原則（記録）

**Why**: 議員業務における市民の声は「対応完了で消える workflow item」ではなく「議員の活動履歴・市民との関係性の記録」であるべき。「未着手→完了」という workflow 管理は タスクDB の役割。意見DB はログ。

**How to apply**:
- 意見DBには statusで workflow 駆動するビュー（未着手のみ等）を作らない
- タスクDBで「関連市民意見」relation 経由で「この意見に紐づくタスク」を抽出するビューを作る
- 完了した意見も削除せず、対応メモに完了経緯を記録して残す
