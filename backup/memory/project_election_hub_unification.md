# 地域・選挙ページ階層の統一（2026-05-03）

## 旧状態の問題

2つの並列親が同テーマを管理していた:

**Structure A: ★選挙プロジェクト（プロジェクトDBレコード）配下**
- 地域別活動計画
- 後援会役員・サポートメンバー
- 新年挨拶まわり
- 👣 歩いて聞く 11,420軒
- 📊 投票区4象限分析
- 🎤 みずきが丘市政報告会プロジェクト

**Structure B: 🗺️ 地域・選挙ハブ（朝のダッシュ配下）**
- 地区別関係性メモDB / 9地区訪問DB / 後援会員DB（mention or 実体）

→ **どちらが本流か不明確、戦略コアは★選挙にあり戸別訪問DB集約は地域・選挙ハブ、と分散**。


## 実施内容

1. ✅ 地域・選挙ハブのコンテンツ全面リライト — ★選挙の戦略コア・月次タイムライン・チェックリストを吸収
2. ✅ サブページ6個を地域・選挙ハブ配下へ移動:
   - 歩いて聞く 11,447軒（34ecf503...94f0）
   - 投票区4象限分析（34ecf503...658a）
   - みずきが丘市政報告会プロジェクト（34ecf503...0a80）
   - 後援会役員・サポートメンバー（2ddcf503...33d5）
   - 新年挨拶まわり（2decf503...bd59）
   - 地域別活動計画（2e9cf503...d1bc・廃止DBのみ）
3. ✅ ★選挙ページをスリム化 — 「地域・選挙ハブへ統合済み」の誘導のみ。タスクDB relation 用 shell として保持（プロジェクトDBレコードなので削除不可）
4. ✅ 全体地図 Tier 2 地域・選挙ハブセクションを刷新

## 設計原則

- **★選挙** はプロジェクトDBレコードとして存続（タスク関連付けに必要）。中身は空にして「地域・選挙ハブへ」と誘導
- **地域・選挙ハブ**が**人間が常に開く入口**。すべての戦略・サブページ・DBがここから派出
- 旧「地域別活動計画」配下の廃止DBは引き続き残置（参照保護）
