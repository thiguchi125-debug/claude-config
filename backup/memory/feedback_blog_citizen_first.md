ブログのテーマを **モードA（市民情報伝達）** と **モードB（政策論考）** に分けて書く。両者は**読者と目的が違う**ため、構成も締めも変える。

**Why**: 2026-05-04の防災気象情報ブログで、私（Claude）はモードAテーマ（5月29日の制度変更を市民に届ける）にモードB構造（議会アクション宣言）を機械適用してしまい、「6月議会で5本柱を問います」が前面に出てしまった。結果、市民が知るべきレベル別の意味＋行動（レベル4=全員避難 等）が後景化し、草川による全面改訂で「市民に届ける」基調に書き直された。「議会で問います」は議員業績アピール色が強く、市民情報伝達ブログでは市民の立場・行動を消してしまう。

**How to apply**:
1. blog-writer / content-pipeline 起動時、書き始める前に必ず **モード判定** を行う：
   - 「市民が今すぐ知って動くべき情報」がテーマの中心 → **モードA**
   - 「議員としての立場・主張・市政への問題提起」がテーマの中心 → **モードB**
2. **モードAでは原則として「6月議会で問います」「議会でしっかり取り上げます」を主役にしない**。触れるなら短い言及程度
3. **モードAの第5段は「市民への直接的呼びかけ」を主軸**に：
   - 「届かないをつくらない」「市民に必要な情報を届ける」「家族にも共有してください」等
4. **モードAの第4段「亀山市でできること」も注意**：「議員から市政への要求リスト」になりすぎていないか／市民が取るべき行動が含まれているか
5. 迷ったら草川にASK_USERで「これは市民情報伝達寄り？それとも政策論考寄り？」と確認
6. 「制度はある。でも、届かないなら意味がない」（草川の核心メッセージ）はモードAでこそ生きる
7. キーフレーズ「届かないを終わらせる」より「**届かないを、つくらない**」のほうが穏健で建設的（草川改訂版で採用）
