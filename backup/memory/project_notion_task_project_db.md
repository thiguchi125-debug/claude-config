**フラット2層運用ルール（2026-04-24移行）**

階層は「**プロジェクト → タスク（1対多）**」の2層のみ。自己リレーション（親プロジェクト/サブプロジェクト／親タスク/サブタスク）は**廃止**。

## DB役割分担

- **✅ タスクDB** (`collection://292cf503-a68f-81c6-b9dd-000b3ffdd2ce`, page `292cf503a68f814eafcdec9eed56f273`)
  - 1人が1回やれば終わる具体アクション（30分〜1日）を管理
  - 例：「薄井さんに電話」「チラシ配布」「警察署へ照会」
  - プロパティ：`タスク名`(title) / `ステータス`(inbox/Wish List/Done/Archive) / `優先度` / `期限` / `プロジェクト`(プロジェクトDBへのリレーション) / 相談元氏名・連絡先（市民相談用）
  - **新規登録は必ず `ステータス: "inbox"`**
  - **`親タスク`／`サブタスク` は使わない**（プロパティ自体を削除済み）

- **🗂️ プロジェクトDB** (`collection://292cf503-a68f-81fe-bd40-000b64314f2e`, page `292cf503a68f813aa68dd75cbc469c92`)
  - 複数タスクを束ねる目的・テーマ（週〜年単位）を管理
  - 例：「🌸 コスモス栽培プロジェクト2026」「★選挙」「中部中通学路対応」
  - プロパティ：`プロジェクト名`(title) / `ステータス`(バックログ/進行中/完了/中止) / `優先度` / `期間` / `要約` / `責任者` / `タスク`(タスクDBへのリレーション) / `進捗率`(タスクステータスのロールアップ)
  - **`親プロジェクト`／`サブプロジェクト` は使わない**（プロパティ自体を削除済み）

**Why:** 2026-04-24にユーザーから「プロジェクトとタスクの違いが曖昧」「親子階層が同じDBにあると見づらい」との指摘。自己リレーションとクロスリレーションが4種類絡むと混乱するため、**1種類（プロジェクト→タスク）のみ**に集約。Notionのエージェント作成時に親+サブ階層でタスクを作るとサブが自動でプロジェクトDB側に吸収される不具合も観測された

**How to apply:**
- **タスク=個別アクション／プロジェクト=テーマ束ね** で判別
  - 30分〜1日の単発作業 → タスク
  - 複数タスクを束ねる目的 → プロジェクト
- プロジェクトにぶら下がる複数ステップは、すべて **タスクDBに並列タスクとして** 登録し `プロジェクト` リレーションで親プロジェクトに紐付け（親タスクで階層化しない）
- タスク同士のグルーピングが必要な場合はタイトルにプレフィクス（例：`[海の日] 薄井さんに声をかける`）で表現
- プロジェクト同士のグルーピングは原則しない（必要なら別テーマとしてフラットに並べる）
- スキル／エージェント側の運用：nichijo・citizen-inquiry-responder・policy-update 等で"親タスク＋サブタスク"階層化ロジックは使わない。すべて並列タスク＋プロジェクトリレーションで表現

**移行履歴（2026-04-24）**
- プロジェクトDBから `親プロジェクト`／`サブプロジェクト` プロパティ削除
- タスクDBから `親タスク`／`サブタスク` プロパティ削除
- プロジェクトDB内の「タスクっぽいもの」16件をタスクDBに複製移行、旧版はステータス「中止」化
- 旧日次DB (`b43cf503-a68f-838f-aa23-872cf4b99d92`) はTrash済
