# 議事録参照ルール — 草川発言＋市回答のみ

## ルール
亀山市議会議事録PDF（Drive `1ZEIt8Cq71oYzJ2sJslxuBNI9GlESHYsg` 配下の H30〜R08 各年度フォルダ・公式OneDrive・委員会txt等）を参照する際は、**草川たくや議員の発言**と**それに対する亀山市側の答弁**（市長／副市長／教育長／総務部長／健康福祉部長／市民文化部長／産業環境部長／文化スポーツ部長／こども家庭部長／環境保全室長／議会事務局長等）の**ペアだけ**を抽出して扱う。

他の議員（豊田氏／伊藤氏／前田氏／福沢氏 等）の質問・答弁ブロックは**読み飛ばす**。

## Why
- 草川の制作物（ブログ・SNS・スピーチ・印刷物・一般質問原稿）に他議員の発言が混入すると、誤って「草川の主張」として書き出してしまう事故が起きる。
- voice-dna 抽出時に他議員の声紋が混ざると、声の指紋が汚染される。
- 議事録は1冊数百ページ。全文をコンテキストに載せるとトークンが破裂する。草川パートだけに絞ることで参照効率が桁違いに上がる。

## How to apply
- **Drive PDF を読むとき**: `read_file_content` で取得した本文から、`◯草川議員` `◯草川議員（再質問）` `〇草川議員` 等のセクションを正規表現で切り出し、その直下の市側答弁ブロック（`◯〇〇市長` `◯〇〇部長` 等）までを1ペアとしてセットで保持。
- **ローカル `01_council/*.txt` を grep するとき**: 既に草川分が抽出整理されているものが多い（特に `*_kusagawa.txt` サフィックス付き）。未整理の生PDF由来テキストは「草川議員」キーワードで切り出してから引用。
- **他議員質問への市回答**を引用したい正当な場合（例：類似テーマで他議員が引き出した数字を比較材料として使う）は、必ず**「他議員の質問への答弁を引用している」と明示**してから使う。草川自身の発言としては絶対に書かない。
- **新規 Drive 議事録を `kusagawa_archive/01_council/` に取り込むとき**: 草川パートのみを抽出してから保存し、ファイル名に `_kusagawa_only` サフィックスを付ける。
