# 🏘️ 自治会DB全経路集約強化（2026-05-04）

## 背景

草川「『記録』などあらゆる情報から自治会に関連する内容はこちらにもまとめられるようになっているか？」

旧設計の課題:
- nichijo Step 2.6 が旧地区別関係性メモDB(`520fe503-`)向けで stale
- 検出キーワードが「9地区＋一部町名」のみで48自治会・22地区を網羅していない
- ikenスキル経由の市民意見が自治会DBに紐付かない
- citizen-inquiry-responder も同様

## 修正内容

### 1. nichijo Step 2.6 全面リライト
- 書込先: `520fe503-`（旧）→ `996b1d15-`（自治会DB）
- 検出キーワード: 48自治会名＋22地区名＋施設名リスト
- 活動種別ごとの追記処理:
  - 戸別訪問 → プロパティ加算＋訪問ログ追記
  - 市政報告会 → 自治会DB＋市政報告会DB両方
  - 挨拶回り/SNSやり取り → 関係性メモ「直近の場での発言」追記
  - 主要メンバー判明 → 主要メンバーセクション更新
  - 主要課題判明 → 課題セクション＋プロパティ更新
- Step 2.6-F 多経路対応明記:
  - 「記録: ...」コマンド
  - ikenスキル経由
  - citizen-inquiry-responder
  - SNS/メール由来

### 2. R2分類表 line 1043 更新
- 旧: 🤝 地域関係性系 → 地区別関係性メモDB
- 新: 🏘️ 自治会・地域関係性系 → 自治会別訪問管理DB

### 3. ikenスキル Step 3.5 新設
- 市民意見作成後、自治会名検出→`関連自治会_意見`relation自動設定
- 自治会DB側「直近の場での発言」セクションに要約追記
- 主要課題が新規なら「主要課題メモ」とプロパティも更新

### 4. citizen-inquiry-responder 修正
- Schema keysに`関連自治会_意見`relation追加
- 自治会自動紐付けの動作を明記
