# おやすみシステム

2026-04-26 構築。「おはよう」スキルの夜版として、ユーザーが「おやすみ」と発話するだけで1日の活動を完全自動で集約・整理し、明日へバトンを渡す就寝時オートメーション。

**Why:** 1日の終わりに手動で振り返り・記録・繰越しをすると属人化・取りこぼし・継続性低下が起きる。完全自動化で「夜：集約 → 朝：確認・着手」の対称構造を作り、ohayoとセットで運用する。

**How to apply:**
- 「おやすみ」「おやすみなさい」「good night」「夜のまとめ」「就寝」「1日の振り返り」「夜のブリーフィング」等の発話で `oyasumi` スキル起動
- スキルは **完全自動化**：途中で選択肢・確認を一切求めない（金曜の週次レポートも自動）
- 翌朝 `ohayo` スキルが「📔 昨夜のまとめ」セクションを朝のダッシュボードから読み込む流れを想定

- ohayoと同ディレクトリに配置（自動でスキル登録される）

- 機械集約専用（手動記録の日次ルーティンログDBとは別物）

- ohayoはこれを読んで翌朝の判断材料に使う

## 9-Step 処理フロー

| Step | 内容 | タイミング |
|------|------|-----------|
| 0 | 日付・曜日判定（金曜なら週次レポート有効化） | 毎晩 |
| 1 | 並列データ収集（Calendar明日/今日 + Gmail + Notion DB群） | 毎晩 |
| 2 | nichijo 自動補完（手動記録なき場合のみ、推測ログを `⚠️AI自動生成` で保存） | 毎晩 |
| 3 | 未完了タスクの自動繰越し（期限=今日→明日／期限超過→`⚠️`プレフィクス） | 毎晩 |
| 4 | 未対応市民相談集計（参照のみ） | 毎晩 |
| 5 | 政策候補ドラフト自動抽出（市民意見3+ or ニュース亀山★4+ 3件 のときのみ） | 毎晩 |
| 6 | 📔デイリーサマリページ作成（必須・データ0件でも作る） | 毎晩 |
| 7 | 📅週次レポートページ作成 | **金曜のみ** |
| 8 | 朝のダッシュボードへ「📔昨夜のまとめ」セクション差込 | 毎晩 |
| 9 | チャット出力（簡潔・対話なし） | 毎晩 |

## 完全自動化ルール（厳守）

1. 途中で選択肢・確認・質問を一切しない
2. 失敗時もスキップ続行（fail-safe）
3. 判断保留はドラフト保存（朝確認に統一）
4. Step 6 のサマリ作成は必須（0件でも空サマリ）
5. 数字は実数そのまま（恣意的な丸め禁止）
