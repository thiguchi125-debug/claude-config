# 会議ハブ刷新（2026-05-02）

## 旧状態の問題

1. ハブページが情報過多で何から始めればいいか分からない
2. ミーティングノートDB(`26d7848d-`) のビューが Default のみ → 実用フィルタなし
3. 会議体マスタ(`46414643-`) も Default ビューのみ
4. Notion AI Meeting Notes 機能との連動が「AI元ノートURL に貼る」とだけ書かれていて使い方がイメージできない

## 改善内容

### ミーティングノートDB に6ビュー追加
- 📅 カレンダー (calendar by 開催日)
- 🔥 開催日順（時系列）(table, 開催日DESC)
- 📋 要フォロー (table, 開催日DESC・手動でフォローアップ要否ONフィルタ追加可)
- 🏷️ 種別ボード (board GROUP BY 会議種別)
- ⭐ 重要会議 (table, 重要度=高フィルタ)
- 📦 未分類棚卸し (table, 会議種別=未分類フィルタ)

### 会議体マスタに2ビュー追加
- ▶️ アクティブな会議体 (table, アクティブ状態=継続中フィルタ・直近開催DESC)
- 🏷️ 種別ボード (board GROUP BY 会議種別)

### ハブページ全面リライト
- **⚡ クイックアクション**: 上部に📅カレンダーをinline表示（タブで6ビュー切替）
- **✏️ 新規記録3方法**: AIミーティング/nichijo/手動 を比較
- **📐 標準テンプレート**: 議題/決定事項/アクション/ディスカッション/次回/関連 のフォーマット
- **🏛 会議体マスタ**: ▶️アクティブビューをinline表示
- **🔗 自動連携テーブル**: 5DB(タスク/政策候補/市民意見/一般質問ネタ/AIノート) との relation 一覧
- **🟦 会議種別ガイド**: 4分類＋未分類の判断基準
- **⚙️ 運用ルール**: 6項目に整理
- **📚 既存ページ**: 折り畳みで縮約

## Notion AI Meeting Notes 機能の使い方（明文化）

1. ミーティングノートDB で「+ 新規」→ ページ内で `/AI ミーティング` または マイクアイコン
2. 録音開始 → Notion AI が自動文字起こし＋要約
3. 完了後、要約ページの URL を 📅DB の **AI元ノートURL** にペースト
4. プロパティ（会議種別／会議体／参加者／重要度／開催日）を埋める

## 制約・注意

- ステータス型 (status) の filter が DSL で書き込めない既知バグのため、要フォロービューには「フォローアップ要否=true」フィルタを書き込めなかった。Notion UIで手動追加（一度設定すれば保存）
- ミーティングノートDB の主要トピック (multi_select) は options 空のまま（実運用で増やす）
- 既存DBに対する linked DB view 埋め込みは markdown の `<database url="...?v=<view_id>" inline="true">` タグで実現
