# 2026-05-02 新設DB群

## 👥 後援会員DB
- **collection_id**: `04cc3a1c-f59a-409d-8406-ff4de19eea45`
- **DB URL**: https://www.notion.so/15deb49eddc24e30a89185a932d16ac5
- **親**: 🗺️ 地域・選挙ハブ
- **会員ID**: KAI-{n}
- **主要フィールド**: 氏名/フリガナ/区分(正会員・準会員・支援者・関係者)/関係性/地区(11区分)/電話/メール/住所/誕生日/入会日/最終接触日/接触履歴/重要度(S/A/B/C)/再訪要(checkbox)/9地区訪問リンク(relation)/個人情報配慮(checkbox)
- **設計意図**: 9地区訪問DBと相互参照する「再訪リスト」として活用。重要度S＝最重要連絡先

**Why**: 2026-10-25市議選準備のため、戸別訪問の中で「コア後援者」と「初対面新規」を区別する必要があった。9地区訪問DBが地区単位の進捗管理だけだと個人レベルの追跡が漏れる。

**How to apply**:
- 新規後援者・支援者は本DBに登録
- 接触履歴は日付＋出来事を箇条書きで蓄積（接触頻度の可視化）
- 戸別訪問前に「再訪要」チェック済を地区別フィルタで抽出→重点訪問先リスト
- 個人情報配慮チェックの先は閲覧時に注意

## 📜 公約・実績マッピングDB
- **collection_id**: `4bce7346-9cef-4382-9099-d60b9acb169d`
- **DB URL**: https://www.notion.so/c866a30da4924922998330c2ce04779d
- **親**: 🎯 政策アップデートハブ
- **公約番号**: KOY-{n}
- **主要フィールド**: 公約/カテゴリ(9区分)/任期(1〜3期目)/進捗(未着手→調査中→質問中→部分実現→完全実現/継続中/断念)/実現度(percent)/開始日/期限/市民との約束(What)/現状(Where)/次の一手(How)/関連政策(rel→政策候補DB)/関連質問(rel→一般質問ネタDB)/関連ミーティング(rel→ミーティングノートDB)
- **設計意図**: コンパス3軸とは別軸で「具体的な約束→現状→次の一手」をトラッキング

**Why**: 4年任期内の公約進捗を時系列で可視化する基盤がなかった。選挙時の実績資料・反論材料として必要。「鈴鹿川に遊べる河川敷」のような個別公約が政策候補DBに散在し、現状が見えない。

**How to apply**:
- 公約掲げる時点でDB登録（任期＝3期目(2026-2030)）
- 政策候補DBで具体化した時点で関連政策に紐付け
- 一般質問化したら関連質問にリレーション
- 議会で答弁出たらミーティングノートにリレーション
- 実現度% は感覚値で月次更新（政策コンパス棚卸し時セット）
