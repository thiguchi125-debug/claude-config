2026-05-01に新設。草川指示「複数タスクに渡っている事案はプロジェクト化、今後のタスク登録時も同様に」を恒久ルール化。

## 判定基準A〜E（草川承認済み・2026-05-01）
- **A**: 同一テーマで関連タスクが2件以上集まっている（※2026-04-24に親子廃止のためフラット運用）
- **B**: 同テーマで散在タスク3件以上
- **C**: 着手〜完了が2週間超
- **D**: 複数ステークホルダー絡み（市民・所管課・議会等）
- **E**: 「〜検討」「〜整備」「〜推進」等の継続性ある語感

A〜Eのうち1つ以上該当でプロジェクト化候補。最終決定は草川。

## スキーマ拡張（2026-05-01実施）
### タスクDB(292c_c)
- `昇格判定タグ` MULTI_SELECT(A:red, B:orange, C:yellow, D:green, E:blue)
- `昇格判定日` DATE — 重複判定回避フラグ（同日にnichijo整理→oyasumi繰越がダブル提案しないため）

### プロジェクトDB(292c_fe)
- `由来` SELECT(既存タスク群統合:blue, 新規企画:green, 市民意見起点:orange, 議会案件:red, その他:gray)
- 既存「責任者」「期限」「期間」プロパティで担当・目標期日は対応可能

**Why:** プロジェクト化作業の事後分析（基準別の精度・由来別の進捗率）を可能にし、運用ロジックを学習更新できるようにするため。
**How to apply:** プロジェクト化時は必ず両プロパティをセット。後で「昇格判定日」が当日なら判定スキップ。
