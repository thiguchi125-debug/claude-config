# 🏘️ 自治会別訪問管理DB（2026-05-03）

## 経緯

旧設計の問題:
- **9地区進捗DB**(地区粒度11レコード) と **地区別関係性メモDB**(投票区29分・実データ0件) が並列存在
- どちらも「内容がわかりにくく進捗管理ほぼできない」（草川判断）
- 地区粒度（10地区）では細かすぎず、投票区粒度（29分）では使いにくい

→ 自治会単位（推定100〜150自治会）に細分化＋関係性＋訪問進捗＋市政報告会を統合管理

## スキーマ（30フィールド）

### 基本情報
- 自治会名 (title) / 自治会ID (auto JK-{n})
- 地区 (select: 10地区＋市外＋未分類)
- 投票区 (rich_text) — 旧29分情報の保持用
- 世帯数 (number)
- 自治会長 (text) / 副会長 (text) / 主要連絡先 (text) / 主要メンバー (text)

### 関係性
- 関係性温度 (select: 強支持/支持/中立/要注意/未評価)
- 主要課題 (multi_select: 通学路/公共交通/水道/防災/空き家/高齢化/子育て/道路/農業/その他)

### 訪問進捗
- 訪問予定月 (select: 5月/6月/7月/8月/9月/完了/対応外)
- 訪問予定週 (date) / 訪問完了日 (date)
- 訪問ステータス (status: 未着手/進行中/完了)
- 訪問済軒数 / 在宅対面 / 在宅拒否 / 不在投函 / 後援会入会 (number)
- **達成率** (formula): `if(prop("世帯数") > 0, prop("訪問済軒数") / prop("世帯数"), 0)`

### 市政報告会（Q4 進捗指標2つ目）
- 市政報告会開催済 (checkbox)
- 市政報告会日 (date)
- 市政報告会メモ (rich_text)

### 履歴・関連
- 過去の約束 / 直近の場での発言 / メモ (rich_text)
- 関連タスク / 関連市民意見 / 関連ミーティング (DUAL relation)
- 登録日 / 更新日 (auto)

## ビュー（5本）

- 🏘️ 地区別ボード — GROUP BY 地区
- 📅 月次プラン — GROUP BY 訪問予定月
- 📅 訪問カレンダー — CALENDAR BY 訪問予定週
- 🎤 市政報告会開催済 — FILTER 市政報告会開催済=true
- 📊 全自治会テーブル — SORT BY 地区, 自治会名

- **主指標②**: 市政報告会開催済自治会数
