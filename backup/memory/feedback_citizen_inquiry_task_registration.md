citizen-inquiry-responderは「返信案3パターン＋Notion意見保存＋現状メモ」の3点セットで完結させるのではなく、**返信で約束した行動をNotion日次/継続DBへのタスク登録候補として提示→ユーザー同意後に登録→市民意見リスト側の対応状況も"関係課確認中"に更新**まで1パスで実行する。

**Why:** 2026-04-23、市民からの違法石積み通報への返信案作成で、「明日土木課に電話」「写真受領依頼」「現地確認」など具体的次アクションが出たのに、タスク化されないまま返信作業が終わると実行忘れや取りこぼしが発生する。議員が「言ったこと」を管理するためにはタスクDB連携が必須。

**How to apply:**
- 返信案STEP 4の後に**STEP 5: 次アクションのタスク化提案**（必須・確認あり）を実行
- 単発タスク=✅日次DB (`b43cf503-...`) / 複数ステップ連鎖=📋継続DB親子 (`292cf503-...-c...`) の判断はnichijoと同じフロー
- 提示時はタスク番号付きで列挙し「yes / 番号指定 / no」で選択可能に
- 登録後は市民意見リストページを `mcp__claude_ai_Notion__notion-update-page` で `対応状況: 関係課確認中` に更新
- 議会アクション深掘りはSTEP 6にずらす（任意のまま）
