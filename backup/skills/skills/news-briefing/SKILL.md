# news-briefing — 朝のニュースブリーフィング

草川たくや（亀山市議会議員）が市議会議員として知っておくべきニュースを毎朝まとめるスキル。Notion📰ニュースDBに蓄積し、議会活用視点付きで提示する。

## 起動モード

ユーザーの発言や呼び出し元に応じて2モードを切り替える。

| モード | トリガー例 | 内容 | 所要時間 |
|---|---|---|---|
| **A: 軽量モード**（デフォルト） | 「ニュース教えて」「今日のニュース」「ニュースまとめて」、cron自動実行、ohayoから委譲 | 6カテゴリ各2〜3本、見出し＋一行要約、Notion DB保存 | 約1〜2分 |
| **B: 深掘りモード** | 「〇〇について深掘り」「△△のニュース詳しく」「議会活用視点で」 | 指定カテゴリ・テーマを深掘り、概要300字＋議会活用メモ詳細 | 約2〜3分 |

判別: 「深掘り」「詳しく」「もっと」「議会で使える」「活用視点」等のキーワード or 特定テーマ指定があればB、無ければA。

## 出力先（重要）

**A・B両モードとも、必ず以下の3箇所に書き込む。スキップ禁止。**

1. **Notion📰ニュースDB** （`data_source_id: 29e5c1a2-d64d-4822-81fd-0d642c3f07bc`）
   - 1ニュース＝1ページとして登録
   - プロパティ: 見出し / 日付 / カテゴリ / 関心テーマ / 情報源 / URL / 概要 / 亀山関連度 / 議会活用メモ / ステータス（新着）

2. **Notion 🗞️毎朝ニュースまとめページ**（`page_id: 75681023203042838ff7ed3b99b1ae2d`）
   - 当日分のMarkdown整形版を該当日付セクションに **追記**
   - 既に同日のセクションがあれば上書き

3. **stdout**
   - 概要サマリ（後述「stdout出力フォーマット」に従う）

## カテゴリと収集ルール

以下6カテゴリを **WebSearch + WebFetch を並列で** 横断収集する。

### ① 亀山市・三重県
- **行政・政策系検索クエリ例**: `亀山市 site:city.kameyama.mie.jp`, `三重県 政策 site:pref.mie.lg.jp`, `亀山市 中日新聞`, `三重 伊勢新聞 最新`, `NHK 三重 ニュース`
- **地域事件・事故・安全系検索クエリ例**（在名民放系を含めて必ず併走させる）: `亀山市 site:news.yahoo.co.jp`, `亀山市 東海テレビ`, `亀山市 CBCテレビ`, `亀山市 メ〜テレ`, `亀山市 中京テレビ`, `亀山市 三重テレビ`, `亀山市 火災`, `亀山市 事件`, `亀山市 事故`, `亀山市 救急`, `関町 OR 井田川 OR 加太 OR 昼生 ニュース`, `鈴鹿亀山消防組合`
- **目標**: 2〜4件（うち**最低1件は地域事件・事故・安全系の検索枠を必ず踏んだ上で選別**。該当事案がない日のみ行政・政策系で2〜4件を埋める）
- **重視**: 亀山市議会・市政発表・近隣市町（鈴鹿・津・伊勢）の議会動向に加え、**人命に関わる事件・事故・火災・災害・行方不明・水害等は政策化の容易さを問わず必ず収集対象に含める**

### ② 国政・自治体（地方自治全般）
- **検索クエリ例**: `総務省 地方自治 最新`, `地方創生 2026`, `自治体DX 最新`, `市議会 議員 動向`
- **目標**: 1〜2件

### ③ 関心テーマ（草川の重点6分野）
草川が一般質問で扱う6分野。**毎回1〜2件ずつローテーションで均等にカバー**する（毎日全6テーマを出す必要はない、1週間で全テーマを巡回するイメージ）。

| テーマ | 主な情報源 | 検索クエリ例 |
|---|---|---|
| 子育て | 子ども家庭庁、厚労省、自治体先進事例 | `子ども家庭庁 最新`, `保育 待機児童 2026`, `子育て支援 自治体` |
| 教育 | 文科省、スポーツ庁、教育委員会動向 | `文科省 最新`, `GIGAスクール 2026`, `小中学校 統廃合` |
| 医療健康 | 厚労省、医療政策 | `厚労省 最新`, `地域医療 過疎`, `産後ケア 自治体` |
| 都市計画まちづくり | 国交省、立地適正化、空き家 | `国交省 まちづくり 最新`, `空き家 自治体`, `立地適正化計画` |
| 防災 | 内閣府防災、消防庁、気象庁 | `防災 自治体 最新`, `南海トラフ 三重`, `避難所 マニュアル` |
| AI | 総務省AI、経産省、自治体DX | `自治体 AI 活用`, `生成AI 行政`, `AI教育 最新` |

- **目標**: 合計2〜4件（1日1〜2テーマでOK、週単位でローテ）

### ④ 全国時事（議員として知るべき主要ニュース）
- **検索クエリ例**: `主要ニュース 2026年4月25日`, `朝日新聞 トップ`, `日経 政治 経済`
- **目標**: 1〜2件
- **重視**: 政治・経済・社会の主要ヘッドライン（市政に関わる文脈で語れるもの）

### ⑤ 省庁発表（横断スキャン）
- **対象省庁**: 子ども家庭庁 / スポーツ庁 / 厚生労働省 / 文部科学省 / 国土交通省 / 総務省
- **検索クエリ例**:
  - `子ども家庭庁 報道発表 [今日の年月] site:cfa.go.jp/press`（一覧URL: https://www.cfa.go.jp/news）
  - `文科省 報道発表 [今日の年月] site:mext.go.jp/b_menu/houdou`
  - `厚労省 報道発表 [今日の年月] site:mhlw.go.jp/stf`
  - `国交省 報道発表 [今日の年月] site:mlit.go.jp/report/press`
  - `スポーツ庁 報道発表 site:mext.go.jp/sports`
  - `総務省 報道資料 [今日の年月] site:soumu.go.jp/menu_news`
- **重要**: 一覧ページ（index）ではなく **個別記事URL** をWebFetch対象にする。一覧ページは内容スカスカで取得失敗することが多い
- **目標**: 1〜2件（直近1週間の新着のみ）
- **注意**: 関心テーマ③と重複する場合は③に統合してOK

### ⑥ SNSトレンド（X）
- **検索クエリ例**: `亀山市 OR 三重県 site:x.com`, `Twitter トレンド 政治 三重`, `市議会 X 話題`
- **目標**: 1〜2件
- **抽出基準**: 亀山市・三重県・市政に関連する話題、または草川の関心テーマで話題化しているもの
- **注意**: 個人の批判ツイート等は除外。話題性・トレンド性のあるものに限定

## 実行フロー

### Step 1: 並列リサーチ（WebSearch / WebFetch を同時発火）

6カテゴリの検索クエリを **1つのメッセージで並列に** 投げる。所要時間を圧縮するため絶対に逐次実行しない。

```
（並列実行）
- WebSearch: 亀山市・三重県関連
- WebSearch: 国政・自治体
- WebSearch: 関心テーマ（その日の重点1〜2分野）
- WebSearch: 全国時事
- WebSearch: 省庁発表
- WebSearch: SNSトレンド
```

検索結果から有望なURLを WebFetch で詳細取得（並列）。

### Step 2: 取捨選択・要約

各カテゴリから以下基準で **合計5〜10件** を選別:
- 24〜48時間以内の新着である
- 草川の議員活動（一般質問、SNS発信、市民相談）に活用できる
- 重複報道は最も詳しい1本に集約
- 亀山市との関連度を5段階（★1〜★5）で判定
- **【人命・地域安全 優先確保ルール】亀山市内・三重県内で発生した人命関連・地域安全事案（火災・事件・事故・救急・水害・土砂災害・行方不明・救助案件等）が直近48時間以内に配信されている場合、★★★★★相当として必ず1件以上拾う**。政策化が即座に難しい段階であっても、議会活用メモには「事実関係を整理した上で防火・防災・救急・地域安全の観点で問題提起」「続報モニタリング後に一般質問化を検討」等を記載する。
- **【行政アジェンダ偏重の防止】**「政策・条例・パブコメ・議会発表」だけで5〜10件が埋まらないよう、上記の人命・地域安全枠を優先的に確保する（2026-04-27の関町小崎住宅火災取りこぼし事案を踏まえた恒久ルール）。

各記事について以下を生成:
- **見出し**: 60字以内
- **カテゴリ**: 上記6カテゴリから1つ
- **関心テーマ**: 該当する6テーマからmulti-select（無くても可）
- **情報源**: 中日新聞三重 / 伊勢新聞 / NHK三重 / 亀山市公式 / 三重県公式 / 各省庁 / 全国紙 / X(SNS) / その他から1つ
  - **在名民放（東海テレビ・CBC・メ〜テレ・中京テレビ・三重テレビ）／Yahoo!ニュース等は当面「その他」扱い**で、概要本文の冒頭に `【〇〇配信】` と明記する（例: `【東海テレビ配信】2026年4月26日朝6時半ごろ…`）。Notion DBの選択肢拡張が必要になった時点で別途検討する。
- **URL**: 一次ソースのURL
- **概要**: モードA=80字、モードB=300字
- **亀山関連度**: ★1〜★5
- **議会活用メモ**: 「一般質問でこう問える」「SNSでこう発信できる」「現場視察に繋げられる」など、草川の議員活動への落とし込み方を1〜2文（モードBは詳細）

### Step 3: Notion📰ニュースDBへ書き込み

`mcp__claude_ai_Notion__notion-create-pages` を1回の呼び出しで全件まとめて投入する。

- **parent**: `data_source_id: 29e5c1a2-d64d-4822-81fd-0d642c3f07bc`
- **各ページのプロパティ**:
  - `見出し` (title): 見出し
  - `date:日付:start`: 今日の日付（YYYY-MM-DD）
  - `date:日付:is_datetime`: 0
  - `カテゴリ` (select): 6カテゴリのいずれか
  - `関心テーマ` (multi_select): JSON配列
  - `情報源` (select): 上記情報源リストのいずれか
  - `userDefined:URL` (url): 一次ソースURL
  - `概要` (text): 80〜300字
  - `亀山関連度` (select): ★1〜★5
  - `議会活用メモ` (text): 1〜2文
  - `ステータス` (select): `新着`

### Step 4: 既存ページへMarkdownレポートを追記

`mcp__claude_ai_Notion__notion-update-page` で `🗞️毎朝ニュースまとめ` ページに当日のセクションを追加する。

- **page_id**: `75681023203042838ff7ed3b99b1ae2d`
- **command**: `update_content`
- **追記する形式**:

```markdown
### YYYY-MM-DD
# 📅 政策リサーチ＆一般質問アイデア（YYYY年MM月DD日／朝のリサーチ）

## 🗞 PART 1：ニュースダイジェスト（本日の注目N件）

### ① 【カテゴリ／情報源】見出し
- **概要**: 概要本文
- **情報源**: [タイトル](URL) ／ [タイトル2](URL2)
- **亀山関連度**: ★★★★☆
- **一言コメント**: 議会活用メモ本文

### ② 【...】...
（以下、件数分繰り返し）

## 🎯 PART 2：今日の議会活用視点（モードBのみ）
（深掘りモードの時のみ、3〜5件のテーマ別に「一般質問の問い立て」「SNS発信案」「現場視察候補」を記載）
```

**old_str の取り方（重要・落とし穴注意）**:

⚠️ **絶対に使ってはいけない old_str**:
- `### 履歴` ─ ページ冒頭にあるが、過去レポート内に「履歴」という単語が再出現してマッチが複数になりやすい
- `### YYYY-MM-DD` ─ 同日が既存履歴に存在すると重複マッチ
- 単独の `---` や `## 🗞 PART 1` ─ 過去レポートに何度も登場

✅ **推奨の old_str（一意性が高い）**:
- `- 形式：要点3〜7個＋重要リンク` ─ ページ冒頭の「今日のニュースまとめ」セクション内の固有行。1箇所のみ存在することが確認済み。
- これを old_str にして、new_str に「同じ行 + 改行 + 新セクション全文」を入れることで、最新版がページ冒頭（履歴の上）に挿入される

✅ **new_str の構造例**:
```
- 形式：要点3〜7個＋重要リンク

### YYYY-MM-DD news-briefing
# 📅 政策リサーチ＆一般質問アイデア（YYYY年MM月DD日／朝のリサーチ）

[本日のセクション全文]
```

**もしマッチエラーが起きたら**:
- エラーメッセージ「Multiple matches found for X. Found N matches.」の場合、old_str をより長く・より具体的にする（前後の固有行を含める）
- それでも解決しない場合は、`mcp__claude_ai_Notion__notion-fetch` でページの最新冒頭を取得して、確実に存在するユニークな1行を選び直す
- 最後の手段として page 追記をスキップしDB登録のみで完了報告する（ユーザーに「ページ追記失敗」と明示すること）

**同日セクションの扱い**:
- 既に当日 news-briefing セクションがあれば、その見出し（`### YYYY-MM-DD news-briefing` を含む完全文字列）を old_str にして全体を新版で置換

## 注意事項

- **並列実行を徹底**: WebSearch / WebFetch / Notion書き込みは可能な限り単一メッセージで並列発火する
- **存在しないURLは絶対に書かない**: 必ず WebFetch で実在確認したURLのみDBに記録
- **WebFetchで内容スカスカが返ったらスキップ**: 一覧ページや動的生成ページは内容取得できないことがある。「実質的な情報がない」「ページタイトルのみ」のレスポンスはそのURLの記録を諦め、別の個別記事URLに切り替える
- **重複防止**: 同日に同じ見出し・URLのレコードが既にあれば書き込みをスキップ
  - 開始時に `mcp__claude_ai_Notion__notion-search` で当日分を `data_source_url: collection://29e5c1a2-d64d-4822-81fd-0d642c3f07bc` 内検索し、既存件数を把握する
- **関心テーマのローテーション**: 7日間で6テーマ全てを巡回することを意識（曜日ローテ目安: 月=子育て・教育、火=医療健康、水=都市計画、木=防災、金=AI、土日=亀山市政重視）
- **ニュースが少ない日**: 各カテゴリ最低1件出ることを目指すが、無理に水増しせず実態を反映する。0件のカテゴリは「該当なし」と表示
- **モードA→Bの引き上げ**: ユーザーが「もっと詳しく」「議会活用視点で」と言ってきたら、既にDB保存した当日分のうち指定されたものについてモードB相当に拡張して再表示する
- **cron自動実行時**: stdoutは標準出力ではなくログとして保存（書き込みのみが目的）。ohayoスキルが後でDBから読む
