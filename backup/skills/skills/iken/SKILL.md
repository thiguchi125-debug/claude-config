# 市民意見コピペ受付スキル（iken）

草川たくや宛にあらゆるチャネルで届いた市民意見テキストをユーザーがコピペするだけで、(1) Notion📋市民意見受付BOXへ自動記録、(2) 返信案3パターン生成、(3) 次アクションのタスク化提案、までを一気通貫で行う秘書スキル。

GAS等の自動連携は使わず、人手コピペ起点の**手動ハイブリッド運用**。

## 起動トリガー

ユーザー発言に以下のいずれかが含まれたら必ず起動：

- 「意見届いた」「ご意見届いた」「届いた意見」
- 「意見コピペ」「市民意見コピペ」「相談コピペ」
- 「市民から意見」「市民から相談」
- 「フォーム回答届いた」「フォームに回答」
- 「DMで意見」「DMで相談」「DMが来た」
- 「メールで意見」「メール届いた（市民）」
- 「LINE意見」「LINEで相談」
- 「手紙届いた」「窓口で意見」「電話で意見」

**重要：「記録」ワード単体は使わない**（nichijoスキルとの衝突回避のため）。

## Step 1: コピペ内容の受領と解析（30秒以内）

ユーザーがコピペしたテキストから以下を抽出：

1. **氏名**：明記があれば実名、なければ「匿名」
2. **連絡先**：メールアドレス（正規表現で抽出）、LINE ID、電話番号、その他。複数あれば全て
3. **本文**：意見・要望本体
4. **経路**：以下の手がかりで推定
   - 「お名前」「メールアドレスまたはLINE ID」「ご意見」等のラベル → **フォーム**
   - メールヘッダ風（From: / Subject: / 「下記の件で〜」等）→ **メール**
   - 「DMで」「Twitter」「Instagram」「Threads」「Facebook」等 → **SNS DM**
   - 「LINEで」 → **LINE**
   - 「電話で」「もしもし」「お電話いただき」 → **電話**
   - 「窓口で」「事務所に来られた」 → **窓口**
   - 「お手紙」「封書」 → **手紙**
   - 推定不能 → ユーザーに**1問だけ確認**：「経路を教えてください（フォーム/メール/SNS DM/LINE/電話/窓口/手紙/口頭/その他）」
5. **カテゴリ**：道路/防災/福祉/子育て/教育/環境/公共交通/行政手続/その他から複数選択
6. **地区**：本文中の地名（亀山/関/井田川/川崎/昼生/神辺/白川/野登/加太）から推定。なければ未記入
7. **緊急度**：以下の4-0判定基準で
   - **高**：被害発生中／危険／継続的問題／時間制約あり
   - **中**：困りごと・要望（明確な被害なし）
   - **低**：軽い気づき・感想・提案

## Step 2: 件名と次アクションの生成

- **件名**：意見冒頭60字 + 「（〇〇様）」（匿名なら「（匿名）」）
- **次アクション**：基本「返信送付」、要対応なら「担当課確認 → 返信」等

## Step 4: 返信案を3パターン生成（このステップが本体）

### 4-0: 期待アクションレベル判定

返信のコミットメントレベルは相談者のトーンと期待アクションレベルにキャリブレーションする：

| レベル | 相談者トーン例 | 適切な約束の上限 |
|--------|----------------|----------------|
| **L1: 軽い気づき・提案** | 「一度考えては」「〜と思います」「ご検討いただければ」 | 担当課に**実情を伝える／申し入れる**まで |
| **L2: 困りごと・要望** | 「困っている」「対応してほしい」「直してほしい」 | 担当課**確認・照会＋経過報告**。場合により現地確認 |
| **L3: 切実な訴え・被害発生** | 「危険」「被害が出ている」「許せない」 | 現地確認・担当課への是正要請・必要なら**議会質問化** |
| **L4: 制度的論点／草川が議会化したい** | 草川自身が議会化を明示／長年の構造課題 | **議会質問化を明確に約束**。松竹梅提案も提示 |

**デフォルトはL1〜L2と推定（軽め寄り）**。L3以上は明確な根拠が読み取れる場合のみ。L1/L2案件で**勝手に議会質問を約束しない**。

### 4-1: 3パターンの方向性

- **🅰 共感・寄り添い型**（300〜500字）：相手の気持ちへの共感を厚めに、行動も爽やかに約束。熱量はベースとして保ったまま語り口だけを柔らかく
- **🅱 行動宣言型**（400〜600字）：共感は簡潔に、適切な範囲の具体アクションを明確に約束。熱量MAX。約束する内容はレベル相応
- **🅲 簡潔・コンパクト型**（150〜250字）：DM・SNSリプライ想定。熱量を凝縮、絵文字なし

### 4-2: 全パターン共通ルール（厳守）

- **一人称は必ず「私」**（「僕」「俺」禁止）
- **熱量・爽やかさ・若々しさ**を全パターンで効かせる
  - 熱量ワード：「本気で」「必ず」「全力で」「覚悟を持って」「やり切ります」「妥協しません」
  - 爽やかワード：「嬉しい」「ワクワク」「熱くなった」「前向きに」「一緒に」「未来を」
  - 若々しさ：短文リズム、自分の体験や感情、「〜と思います」連発回避
- 冒頭「〇〇様／ご連絡ありがとうございます」（実名不明なら「〇〇様」伏せ）
- 共感 → 現状認識 → **具体行動（期限・場所・相手を明示）** → **未来ビジョン**、の4要素（Cは圧縮）
- 結びは「前向き・未来志向」で締める
- スマホで読みやすい改行（2〜3文ごと、段落間は空行）
- 絵文字なし
- 役所言葉禁止：「〜させていただきます」「〜かと存じます」「遺憾に思います」
- 約束できないことは約束しない
- 署名：「亀山市議会議員 草川たくや」

### 4-3: 経路別の体裁調整

| 経路 | 体裁 |
|------|------|
| メール | 「件名」付き、本文1000字程度OK、署名フル |
| フォーム | 返信先がフォームメアドの場合はメール体裁、不明なら短文＋次連絡導線提示 |
| LINE | 段落少なめ、改行多め、スタンプは使わず文字で温度感 |
| SNS DM | 短文凝縮、改行多め、署名は「草川たくや」のみ |
| 電話 | 折り返し電話用の**話す内容の骨子**として整理（口頭調） |
| 窓口・口頭 | フォローアップ手紙 or LINE等を選択肢として提示 |
| 手紙 | 同じく手紙返信の体裁、季語と署名 |

## Step 5: 次アクション抽出 → ✅タスクDB登録候補提示

返信案3パターンの中から、**今後草川が実行すべきアクション**を抽出して箇条書き提示：

- 例：「担当課（建設部）へ通学路安全について照会（5月7日まで）」
- 例：「井田川地区の現地確認（5月10日週）」
- 例：「6月議会一般質問の論点候補に追加」

各アクションを**✅タスクDB**（`292cf503-a68f-81c6-b9dd-000b3ffdd2ce`）へ登録するか確認。

登録時のプロパティ（タスクDB運用ルールに従う）：
- ステータス：`inbox`（必須）
- タスク名先頭に `[市民意見-CV-N]` プレフィクス（受付IDと紐付け）
- 相談元氏名・連絡先：📋受付BOXページから転記
- 期限：明示があれば

複数ステップなら🗂️プロジェクトDBに親プロジェクト作成→relation紐付けを提案（既存運用準拠）。

## 深掘り依頼への対応

ユーザーが以下のいずれかを言ったら：

- 「詳しく」「深掘り」「もっと調査」「議会で取り上げたい」
- 「政策化したい」「他自治体の事例」「データ追加で」

→ `policy-researcher` エージェントを呼び出して先進事例・国政動向を補強。
→ 「議会質問にしたい」なら `council-material-creator` エージェントへ繋ぐ。

## 重要案件の📝市民意見リスト昇格フロー

📋受付BOXに溜まった意見のうち、草川が**自ら継続対応する重要案件**は手動で📝市民意見リスト（c2c34bd8-）へ転記：

1. 📋受付BOXの該当ページの「既存リストへ移管」チェックボックスをON
2. 📝市民意見リストへ同等プロパティでコピー作成（草川が手動 or ユーザー指示で本スキル外で実行）

📋受付BOX = 受信intake、📝市民意見リスト = 草川が責任もって対応する重要案件、という2層運用。

## エラーハンドリング

- Notion保存失敗：エラー文を提示しつつ、返信案3パターンとメモは出力（記録漏れ警告）
- 経路推定不能：1問だけユーザーに確認
- 内容が短すぎ／意味不明：本人に追加情報を求める返信案を提示
