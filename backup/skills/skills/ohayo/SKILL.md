# おはようスキル — 朝のブリーフィング

ユーザーが朝の挨拶（「おはよう」など）をしたとき、または「今日のタスク」「今日の予定」を聞いたとき、このスキルを実行する。

## やること

以下の3つのデータソースから情報を**並行して**取得し、一気にまとめた朝のブリーフィングを日本語で返す。

### 1. Google Calendar — 今日の予定

`mcp__claude_ai_Google_Calendar__list_events` を使い、今日（現在の日付）の予定を**2つのカレンダーから並行して**取得する。

- calendarId: `t.higuchi125@gmail.com`（プライマリ）と `kusakawa.taku@gmail.com` の2つを同時に取得
- time_min: 今日の00:00:00（ISO 8601形式）
- time_max: 今日の23:59:59（ISO 8601形式）
- 結果をマージして時刻順に並べ、タイトル・開始時刻・終了時刻・場所（あれば）を抽出する

### 2. Gmail — 要チェックメール

`mcp__claude_ai_Gmail__search_threads` を使い、今日届いた未読または重要なメールを取得する。

- query: `is:unread newer_than:1d` または `is:important newer_than:1d`
- max_results: 10
- 件名・送信者・受信日時を抽出し、特に対応が必要そうなものを優先する

### 3. Notion — 朝のダッシュボード＋積み残し確認＋ニュース

以下を**並行して**取得する。

#### 3-0. 📰 ニュースDB — 今朝のニュース
`mcp__claude_ai_Notion__notion-query-database-view` または `notion-search` で本日のニュースを取得する。

- data_source_id: `29e5c1a2-d64d-4822-81fd-0d642c3f07bc`
- filter: `日付` が今日（YYYY-MM-DD）のもの
- 取得するプロパティ: `見出し` / `カテゴリ` / `関心テーマ` / `情報源` / `亀山関連度` / `URL` / `議会活用メモ`
- ソート: `亀山関連度` 降順、同順なら `カテゴリ` 順
- 全件取得（5〜10件想定）

**当日のニュースが0件だった場合**（cronが失敗 or まだ早朝で未実行）:
- news-briefingスキルをライブ実行する代わりに「⚠️ 本日のニュースは未収集です」と表示し、`/news-briefing` で手動実行できる旨を案内する
- フルブリーフィング処理（WebSearch多用）はohayo内では実行しない（重いため別スキルへ委譲する設計）

#### 3-1. 朝のダッシュボード
`mcp__claude_ai_Notion__notion-fetch` でダッシュボードページを取得する。
- id: `https://www.notion.so/722beb9e9827421aa5dbbef67c1c4688`
- 「進行中プロジェクト」の「次にやること」を抽出する
- 取得したページのテキスト全体をStep 4で使うためにそのまま保持しておく

#### 3-1b. 今日のタスク（タスクDBから直接取得）
`mcp__claude_ai_Notion__notion-search` でタスクDBを直接クエリする（ダッシュボードにはリンクしか無いため、DB本体から取得する）。
- data_source_url: `collection://292cf503-a68f-81c6-b9dd-000b3ffdd2ce`
- 取得方針: **期限=今日** または **今日作成** かつ **ステータス ≠ Done / Archive** のタスク全件
- 取得するプロパティ: `タスク名` / `ステータス` / `優先度` / `期限` / `url`
- 出力時は Step 5 の「✅ 今日のタスク」テーブルにそのまま流し込む（リンクだけの案内は禁止）
- **勝手に件数を絞ったり、特定の2〜3件だけをピックアップしたりしないこと**。全件を掲載する

#### 3-2. 一般質問ネタDB — 未整理件数
`mcp__claude_ai_Notion__notion-query-database-view` で未整理ネタ数を確認する。
- data_source_id: `42716725-fece-497f-9782-705076539de4`
- filter: `状況` が `未整理` または `収集` のもの
- 件数のみ取得（タイトル一覧は不要）

#### 3-3. SNS投稿管理DB — 未着手件数＋直近3件
`mcp__claude_ai_Notion__notion-query-database-view` で未投稿の下書き数と直近タイトルを確認する。
- data_source_id: `1bd98deb-624f-402c-aeb3-bdaa4782b389`
- filter: `ステータス` が `未着手` のもの
- **総件数** と **登録日降順の直近3件の投稿タイトル** を取得する
- **禁止事項**: 「2件」「特定のテーマ2つ」など恣意的に件数を絞って提示しないこと。必ず総数＋直近3件という形で出す

#### 3-4. 昨日の活動ログ確認
`mcp__claude_ai_Notion__notion-query-database-view` で昨日のログページの有無を確認する。
- data_source_id: `f6d15ac0-73fa-414f-8242-03e8f06aa4e3`
- filter: `日付` が昨日の日付（YYYY-MM-DD）のもの
- 存在すれば「記録済み」、なければ「未記録」として扱う

#### 3-5. 進行中プロジェクト＋次アクション取得
`mcp__claude_ai_Notion__notion-query-database-view` でプロジェクトDBから進行中プロジェクトを取得する。
- view_url: `https://www.notion.so/292cf503a68f813aa68dd75cbc469c92?v=292cf503a68f814c914e000c76ddc78f`
- 取得後、`ステータス` が `進行中` のもののみフィルタ
- 取得するプロパティ: `プロジェクト名` / `url` / `タスク`（relation） / `期限` / `優先度` / `進捗率`

**表示構造（2026-04-24フラット2層運用）**
- プロジェクトDBはフラット1層。プロジェクト間の親子階層（`親プロジェクト`／`サブプロジェクト`）は廃止済み
- 全プロジェクトを並列（フラット）で表示する

**ソート順**
- `優先度` 降順（高→中→低→未設定）、同順なら `期限` 昇順

**各プロジェクトの「次にやること」抽出**
- タスクDB（`collection://292cf503-a68f-81c6-b9dd-000b3ffdd2ce`）をクエリ
- filter: 該当プロジェクトにリレーションされている かつ `ステータス` が `Done` / `Archive` 以外
- ソート: `期限` 昇順（期限なしは末尾）、同順なら `優先度` 降順
- 先頭1件の `タスク名` を「次にやること」として保持
- タスクがゼロの場合は「次にやること」は `（タスク未登録）`

**禁止事項**:
- 進行中プロジェクトがゼロの場合はセクションを空箇条書きにせず、`- （現在、進行中プロジェクトはありません）` と出力する。勝手に仮プロジェクト名を書かないこと

### 4. Notionダッシュボードへの自動書き込み

**⚠️ この手順は必須。ページが「すでに最新に見える」場合でも必ず実行すること。スキップ禁止。**
**⚠️ チャット出力に表示した重要情報は全てダッシュボードにも反映する。「チャットで言ったから書かない」は禁止。**

ブリーフィング出力後、`mcp__claude_ai_Notion__notion-update-page` で以下の6セクションを一度の `update_content` 呼び出しで更新する。

- page_id: `722beb9e9827421aa5dbbef67c1c4688`
- command: `update_content`

#### old_str の取り方（重要）

**Step 3で取得したページのテキストから、各セクションの先頭行（`## ...`）から次の `##` 行の直前までを正確にコピーして `old_str` に使う。** 自分で推測・生成してはならない。

#### 更新する6セクション（必須）

**セクション1: 今日のスケジュール**

- old_str: ページから取得した `## 📅 今日のスケジュール（...）` セクション全体
- new_str: 以下の形式（Step 1のカレンダーデータで埋める）

```
## 📅 今日のスケジュール（YYYY年M月D日 曜日）
- 終日　イベント名
- HH:MM〜HH:MM　イベント名（場所）
...（取得した全予定を時刻順に）
```

**セクション2: 重要メール**

- old_str: ページから取得した `## 📬 重要メール・要対応` セクション全体
- new_str: 以下の形式（Step 2のメールデータで埋める）

```
## 📬 重要メール・要対応（YYYY年M月D日更新）
- **【送信者】件名** — 対応内容の一行要約
...（広告・定期メルマガ以外の要対応メールのみ）
※上記以外は販促・ニュース配信で返信不要。
```

重要メールが0件の場合: `※本日の要対応メールはありません。` のみを記載。

**セクション2.5: 🎯 今日のフォーカス（毎日新規生成・必須）**

- old_str: ページから取得した `## 🎯 今日のフォーカス` セクション全体（既存があれば）
- new_str: 以下の形式（カレンダー・メール・タスク・積み残しを総合した1日の戦略）

```
## 🎯 今日のフォーカス（YYYY-MM-DD）
**今日の1本軸：[1文で本日の戦略]**
- **午前/午後/夜の動線**：[時間帯別の優先順位と移動経路]
- **本日中に必須**：[期限超過 or 今日中タスクで動かないと致命のもの2〜3件]
- **月曜以降に振替推奨**：[週末で物理的に不可能 or 効率悪いタスク]
- **隙間時間で**：[SNS投稿・写真撮影・ショート動画など]
```

既存セクションがない場合は、`## 📬 重要メール・要対応` の直後に挿入する（old_strに重要メール末尾＋次セクション先頭を含めて挟み込む）。

**セクション2.6: 🗞️ 今朝のニュース（毎日リフレッシュ・必須）**

- old_str: ページから取得した `## 🗞️ 今朝のニュース` セクション全体（既存があれば）
- new_str: 以下の形式（Step 3-0 のニュースDBデータで埋める）

```
## 🗞️ 今朝のニュース（M/D 6:00 JST cron取得・亀山関連度順）
- [カテゴリ][★★★★★] [見出し](記事URL) — 一行コメント（議会活用視点）
- ...（取得した5〜10件全件、亀山関連度降順）
→ 全件は [📰 ニュースDB](https://www.notion.so/f2eefc669dd54648bbcdacdc8afa1158) で確認
```

ニュースDBに本日記事が0件の場合は `⚠️ 本日のニュースは未収集です。/news-briefing で手動取得できます` のみ記載。

**セクション3: 🔗 進行中プロジェクト**

- old_str: ページから取得した `## 🔗 進行中プロジェクト` セクション全体（`→ 全プロジェクト: ...` の行まで含む直前の `##` 行の手前まで）
- new_str: 以下の形式（Step 3-5 のプロジェクトデータで埋める）

```
## 🔗 進行中プロジェクト（[進行中件数]件）
- [プロジェクト名](プロジェクトURL)【優先度：高】（期限：YYYY-MM-DD）
	- **次にやること：** [次アクションのタスク名]
- [プロジェクト名](プロジェクトURL)【優先度：中】
	- **次にやること：** [次アクションのタスク名]
- [プロジェクト名](プロジェクトURL)
	- **次にやること：** （タスク未登録）
→ 全プロジェクト: [プロジェクト＆タスク](https://www.notion.so/292cf503a68f802da262d7e0dab4ebf3) / [プロジェクトDB](https://www.notion.so/292cf503a68f813aa68dd75cbc469c92)
```

表示ルール:
- プロジェクト見出し行: `- [名](URL)【優先度：X】（期限：YYYY-MM-DD）` — 優先度／期限が未設定なら該当パート省略
- 各プロジェクトは必ず「次にやること」を1行持つ（タスクがなければ `（タスク未登録）`）
- 全プロジェクトを並列（フラット）に表示（ネスト禁止）

進行中プロジェクトが0件の場合:
```
## 🔗 進行中プロジェクト（0件）
- （現在、進行中プロジェクトはありません）
→ 全プロジェクト: [プロジェクト＆タスク](https://www.notion.so/292cf503a68f802da262d7e0dab4ebf3) / [プロジェクトDB](https://www.notion.so/292cf503a68f813aa68dd75cbc469c92)
```

**セクション4: 今日のチェックリスト（毎日リセット）**

- old_str: ページから取得した `## ✅ 今日のチェックリスト` セクション全体
- new_str: チェック状態をすべて `[ ]` にリセットして以下を出力

```
## ✅ 今日のチェックリスト
- [ ] 情報収集（ニュース・SNSチェック）
- [ ] コメント・DM返信
- [ ] SNS投稿
- [ ] 挨拶回り・地域活動
- [ ] 議会・行政関連作業
```

#### 4-2. 要対応メールをタスクデータベースに追加

広告・ニュースレター以外で対応が必要と判断したメールを `mcp__claude_ai_Notion__notion-create-pages` で統合タスクDBに追加する。

- parent: `data_source_id: 292cf503-a68f-81c6-b9dd-000b3ffdd2ce`
- タスク名: `【メール確認】送信者: 件名`
- ステータス: `inbox`
- 優先度: 内容に応じて `高` / `中` / `低`
- date:期限:start: 今日の日付（YYYY-MM-DD）
- date:期限:is_datetime: 0

すでに同じ件名のタスクが存在する場合は追加しない（重複防止）。広告・PR・定期メルマガは追加しない。

#### 4-3. ブラウザでダッシュボードを開く

上記の書き込みが完了したら、Bash ツールで以下を実行する：

```bash
open "https://www.notion.so/722beb9e9827421aa5dbbef67c1c4688"
```

## 注意事項

- データ取得は4つのソース（Calendar・Gmail・Notion各種・ニュースDB）同時並行で行い、レスポンスを速くする
- ニュース欄はDB読み出しのみ（軽量）。新規取得が必要な場合はnews-briefingスキルへ委譲し、ohayo内でWebSearchを呼ばない
- メールは件名と送信者のみ表示し、本文は含めない（プライバシー配慮）
- Notionのタスクは完了済みのものは除外する
- 「今日のフォーカス」はデータから機械的に出すのではなく、ユーザーの状況を読んで1〜2文で端的にまとめる
- 全体で画面1スクロール以内に収まるよう簡潔にする（ただし「今日のタスク」テーブルは件数にかかわらず全件表示する — 折り畳みや「他◯件」と省略するのは禁止）
- **Notion書き込み（Step 4）は必ず実行する。「ページが最新そう」「同じ内容になる」などの理由でスキップしてはならない**
- **タスク・SNS下書きの数値は必ずDBクエリの結果そのまま出す**。恣意的に絞り込んだり「代表例として2件」という表現で実数と異なる件数を提示するのは禁止
