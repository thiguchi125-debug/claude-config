# Short Video Virality Playbook

> ≤60秒ショート動画（TikTok / YouTube Shorts / Instagram Reels）の**完視聴率最大化／スキップ回避／拡散誘発**のための設計知識ベース。
> `short-video-virality-architect` エージェントが必読する核心資料。

---

## §1. フック15分類（冒頭0〜3秒の停止トリガー）

スクロール慣性に勝てる手段は2つだけ：**「予測不可能な刺激」**か**「自分事化された問い」**。

### A. 数字ショック型
- 構造: 視聴者の常識から外れた数字を1秒以内に提示
- 例: 「**亀山市、年1,328万円が消えてる**」「**25年続く制度、知ってますか**」
- 強み: 認知負荷低・即理解可
- 注意: 数字が地味だと逆効果。"驚く理由"が3秒以内に分かる必要

### B. コントラリアン型（通説否定）
- 構造: 「みんな〇〇と思ってるけど、違う」
- 例: 「**Kカードを廃止しろ、と思ってませんか？ 違う、進化させる**」
- 強み: 認知的不協和でスクロール停止
- 注意: 政治家利用時は名指し批判禁止、組織抽象（"市役所"）まで

### C. 質問型
- 構造: 1秒以内に答えたくなる問い
- 例: 「**3つのカード、どれが一番税金使ってるか分かります？**」
- 強み: 脳が問いに反応する自動性
- 注意: Yes/Noで終わる質問はNG。「考える余白」がある問いを

### D. パターン中断型
- 構造: 視覚／音声の予測を裏切る
- 例: 0.5秒の無音→突然の声／カメラを地面に置いて画角崩す
- 強み: 脳の「予測誤差検出」に直接刺さる
- 実装: 無音0.3秒→声、ジャンプカット、逆光→正光の急変化

### E. リストプロミス型
- 構造: 「3つの〇〇」「5つの理由」と数を提示
- 例: 「**亀山市の3つのカード問題、ぜんぶ話します**」
- 強み: 完視聴の動機（最後まで見ないと損）
- 注意: 実際にN個を簡潔に並べる構造設計が必要

### F. パーソナル賭け金型
- 構造: 自分の身に起きた／起こした出来事
- 例: 「**議場で、市長を10分問い詰めてきました**」
- 強み: 一人称ナラティブの引力
- 注意: 嘘・誇張は信頼破壊。事実ベースに限定

### G. 内部告発型
- 構造: 「〇〇が言わない」「実は」
- 例: 「**市役所が説明しない、年1,328万円の使い道**」
- 強み: 秘密共有の優越感を視聴者に与える
- 注意: 実際の告発でなければトーンダウン必要、誤解誘導NG

### H. 時間圧型
- 構造: 「今すぐ」「〆切」を提示
- 例: 「**6月議会まで2週間。間に合わせる**」
- 強み: 緊急性の認知
- 注意: 政治家は選挙日言及禁止、議会日程は使える

### I. クリフハンガー型
- 構造: 結論を冒頭で予告、過程を見せる
- 例: 「**この提言、市長は何と答えたか――最後まで見て**」
- 強み: 結末欲求でリテンション維持
- 注意: 結末が肩透かしだと信頼失う

### J. 比較型
- 構造: AとBの対比で考えさせる
- 例: 「**飛騨は82億円。亀山は1,328万円。どっちが回ってる？**」
- 強み: 即座の理解＋議論誘発
- 注意: 不公正な比較は逆効果

### K. "N日試した"型
- 構造: 期間×行動を提示
- 例: 「**亀山市の30地区、全部回ってきた結果**」
- 強み: 努力の可視化、信頼構築
- 注意: 実際にやってないことに使わない

### L. リアクション型
- 構造: 表情・反応だけで開く
- 例: 議事録を読んで驚く顔のクローズアップ→「これ、本当ですか？」
- 強み: 感情の即伝播
- 注意: 過剰演技は萎える、自然な驚き必須

### M. アイデンティティ呼びかけ型
- 構造: 「〇〇な人へ」と特定対象に絞る
- 例: 「**30代の亀山市民へ、これ知ってください**」
- 強み: 該当者の自分事化
- 注意: 対象を狭めすぎるとリーチ減

### N. 権威破壊型
- 構造: 「〇〇は嘘」「教科書は古い」
- 例: 「**昔ながらのポイントカード、もう時代遅れです**」
- 強み: 既存秩序への挑戦
- 注意: 政治家は穏当に、対象は制度・物・概念まで

### O. 視覚サプライズ型
- 構造: 物が落ちる／割れる／変化する瞬間
- 例: 紙のKカードを画面下に落として「これ、変わります」
- 強み: 動き＝注目の自動的引力
- 注意: 安っぽい演出になりがち、文脈と接続必須

## §2. Retention Engineering（秒単位の刺激配置）

完視聴率を上げる唯一の方法は**1.5〜2秒ごとに刺激を変える**こと。脳は同じ画を3秒以上見ると注意を切る。

### 60秒の理想構造

| 区間 | 時間 | 役割 | 設計指示 |
|---|---|---|---|
| **Hook** | 0〜3秒 | 停止トリガー | §1の15分類から1〜複数選択／無音0.3秒→声推奨 |
| **Promise** | 3〜7秒 | 続きを見る理由 | 「最後まで見れば〇〇が分かる」を1文で |
| **Setup** | 7〜15秒 | 前提情報 | 1文1事実、画も1秒ごと変化 |
| **Substance** | 15〜40秒 | 中身 | **1.5〜2秒ごとカット切替**／テロップ／表情変化 |
| **Peak** | 40〜50秒 | 最も刺さる1点 | 数字１つ、フレーズ１つに絞る |
| **CTA + Loop** | 50〜60秒 | 行動誘発＋ループ | 最後3秒で冒頭と接続する仕掛け |

### オープンループ多用ルール

- 中盤で「**この答えは最後に話します**」を挿入
- 視聴者の脳に未完了タスクを置く＝Zeigarnik効果
- ループが3つ以上絡むと視聴維持率が劇的に上がる

### カット変化の最低基準

- **30秒尺**: 8カット以上
- **60秒尺**: 15カット以上
- 同じ画角を2秒以上維持しない（talking head脱却）

### 音声設計

- BGMは全体の-20dB以下、声を絶対に邪魔しない
- 0.3秒の無音を冒頭・転換点に配置
- 効果音は1動画3〜5回、強調点のみ

### テロップルール

- **常駐禁止**: 表示は2〜3秒ごとに切替
- **2行以内**: 一画面に文字を詰めすぎない
- **数字は黄色**: 白×黒縁取り＋強調色１色
- **ジェネリックフォント禁止**: 太ゴシック推奨、装飾フォントは1動画1箇所

## §3. プラットフォーム別アルゴリズム信号

同じ動画でもPFごとに最適化指標が違う。1本作ったら3PF用に微調整するのが正解。

### TikTok For You Page

| 指標 | 重要度 | 設計含意 |
|---|---|---|
| 完視聴率 | ★★★★★ | これが最強。短くて完視聴狙う |
| 再視聴率 | ★★★★ | ループ設計必須 |
| シェア | ★★★ | "送りたい人がいる"設計 |
| コメント | ★★★ | 議論誘発装置を仕込む |
| 保存 | ★★ | 後で見返したい価値 |
| いいね | ★ | 軽量シグナル |

**TikTok特化の設計**:
- 60秒ぴったり狙うより**45〜55秒**が完視聴率最大化
- 最後に「**もう一度見てください**」と明示も有効
- ループ用：最後の画と最初の画を視覚的に同一に

### YouTube Shorts

| 指標 | 重要度 | 設計含意 |
|---|---|---|
| 平均視聴時間 | ★★★★★ | 60秒見せ切る |
| スワイプ率 | ★★★★★ | 冒頭3秒で残存決定 |
| チャンネルCTR | ★★★ | プロフィール誘導 |
| いいね・コメント | ★★ | 補助的 |

**YouTube Shorts特化の設計**:
- **冒頭3秒の刺激量**を最大化（YouTube内蔵性が最も厳しい）
- タイトルとサムネ（縦動画でも初フレームが効く）の合わせ技
- 説明欄にチャンネル誘導リンク必須

### Instagram Reels

| 指標 | 重要度 | 設計含意 |
|---|---|---|
| DM共有率 | ★★★★★ | "送りたくなる"が最強 |
| 保存率 | ★★★★ | 役立ち系コンテンツ強い |
| プロフィール訪問 | ★★★ | フォロワー転換 |
| いいね | ★★ | 補助 |

**Instagram Reels特化の設計**:
- **「友達に送りたい」**動機を作る（驚き／共感／お役立ち）
- 役立ち系（リスト形式）は保存率高
- 投稿テキストでも文脈補強、ハッシュタグ最大10個

## §4. コメント誘発装置

コメント欄が活性化する動画はアルゴリズムに強い。

### 効く仕掛け5種

1. **不完全な情報提示**: 「詳しい数字はコメント欄で」
2. **意見分裂の問い**: 「あなたは賛成？反対？」
3. **答えを聞く**: 「他にどんな制度知ってる？」
4. **間違いを置く**: 故意のミスではなく、複数解釈ある余白を残す
5. **対立軸の提示**: 「Aの立場の人、Bの立場の人、それぞれどう思う？」

### 政治家での注意

- 故意の対立煽動は信頼破壊
- 「賛成・反対」型はOK、「敵を作る」型はNG
- コメント返信を3〜10件は手動で返す（アルゴリズム強化）

## §5. ループバック設計（再視聴を促す最後5秒）

最後の3〜5秒で「もう一度見たい」を作る。

### 効くループ構造3種

1. **視覚同一化**: 最後の画を冒頭と同じフレームに（自然なループ）
2. **言語回収**: 冒頭の問いを最後に答える（「最初に言った〇〇、答えは△△」）
3. **新たな伏線**: 「この続きは次回」で再視聴より続編期待を作る

### 草川の動画での適用例

- **冒頭**: 「年1,328万円、知ってますか？」
- **最後**: 「年1,328万円を、ひとつのアプリへ。」
- 同じフレーズを違う意味で回収＝詩的ループ

## §6. Voice-DNA × Virality バランス

草川の声を保ったまま、バイラル設計に乗せる方法。

### 残すもの（草川らしさ）

- ダブルダッシュ「——」: 1動画1〜2回まで（多用は退屈フィラー化）
- 軸3「学ぶ亀山から、選ばれる亀山へ」: 締めに使うとループ強化
- 誠実トーン: 過剰演技禁止
- 数字を伴う具体性: voice-dnaの核

### 抑えるもの（virality妨害要素）

- 「皆さんこんにちは」: **絶対禁止**（即スワイプ）
- 「〜と思います」連発: 1動画3回以下
- 「頑張ります」CTA: 具体的行動に置き換え（「議会で問います」「LINE登録を」）
- 議員バッジ画面常駐: 最初か最後のみ

### バイラル設計に追加するもの

- 表情変化: 真剣→笑顔→驚き→決意の振れ幅
- 画角変化: バストアップだけでなくクローズアップ・引き
- 屋内屋外B-roll: 議場・街・市民との会話シーン

## §7. 自己診断8軸スコアリング

各バリアント（A/B/C）を1〜10点で採点。8点未満の軸は再リビルド対象。

| 軸 | 評価基準 |
|---|---|
| **掴み力** | 冒頭3秒でスクロール停止確率（10=絶対停まる） |
| **本人らしさ** | 草川voice-dna保持度（10=本人と区別不可） |
| **具体性** | 数字・固有名・場所の密度（10=1秒1事実） |
| **情緒** | 感情の振れ幅（10=笑い・怒り・共感が混在） |
| **媒体感** | PF別アルゴリズム最適化（10=完視聴率予測80%超） |
| **AI臭** | 定型句・予測可能フレーズの少なさ（10=AI生成と判別不可な人間味） |
| **行動喚起** | CTAの具体性と実行容易性（10=画面操作で即行動） |
| **ローカル定着** | 亀山市・三重県の固有性（10=他自治体に流用不可） |

合計64点以上で「公開推奨」、48〜63点で「修正推奨」、47点以下で「リビルド」。
